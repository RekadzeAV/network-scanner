Документ фиксирует подробный список задач разработки на основании согласованного плана.

- [ ] Добавить проверку наличия команд в `PATH` до запуска.
- [ ] Нормализовать ошибки: `not_installed`, `permission_denied`, `timeout`, `network_error`.
- [ ] Добавить fallback сообщения в GUI/CLI без падения потока выполнения.

### 2) CI и инженерные проверки
- [ ] Добавить matrix job для Windows/macOS/Linux с `go test ./...` и `go build ./...`.
- [ ] Подключить линтер в обязательные проверки PR.
- [ ] Добавить smoke-build для GUI и CLI артефактов.
- [ ] Добавить статусные бейджи в `README.md`.

### 3) Тестирование
- [ ] Unit-тесты `internal/nettools` (парсинг, таймауты, ошибки инструментов).

### 4) UX/performance

## Дополнительный backlog развития/реализации (D-трек)

### D-планирование: приоритет, сложность, спринт

| ID | Направление | Priority | Complexity | Sprint |
|---|---|---|---|---|
| D1 | Topology hardening (SNMP + граф) | High | L | D1–D2 |
| D2 | Hardening экспорта топологии | High | M | D2–D3 |
| D3 | GUI hardening результатов | Medium | M | D3–D4 |
| D4 | Управление рисками внедрения | Medium | S | D1 (и поддержка до D4) |

### Спринтовая последовательность D-трека

- **Sprint D1**
  - D4: feature-flags, rollback контур, CI smoke-profile для D-направлений
  - D1: правила дедупликации LLDP/FDB, confidence baseline
- **Sprint D2**
  - D1: explain-данные и fallback без Graphviz
  - D2: schema-check + smoke форматов экспорта
- **Sprint D3**
  - D2: эквивалентность `json`/`graphml`, совместимость с yEd/Gephi
  - D3: GUI smoke и visual baseline для таблицы/карточек
- **Sprint D4**
  - D3: responsive/perf budget/cross-check метрик
  - D4: финальная стабилизация и снятие временных feature-flags по готовности
