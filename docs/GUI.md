# GUI версия - Графический интерфейс

## Описание

GUI версия Network Scanner предоставляет графический интерфейс пользователя для сканирования локальной сети с использованием фреймворка Fyne.

## Структура

```
internal/gui/
├── app.go                    # Основная логика GUI приложения
├── scan_controller.go        # UI-оркестрация стадий/состояний сканирования
├── topology_controller.go    # UI-оркестрация стадий/состояний построения топологии
├── operations.go             # Runtime операций (queued/running/success/failed/canceled)
├── security_view.go          # Security Dashboard (audit + risk signatures)
├── results_view.go           # Рендер таблицы/карточек, фильтры и Host Details Drawer
├── results_analytics_view.go # Явная аналитика в пайплайне рендера результатов
├── results_charts.go         # Аналитика и диаграммы
└── results_model.go          # Модель отображения/сортировки

cmd/gui/
└── main.go         # Точка входа GUI приложения
```

## Компоненты

### internal/gui/app.go

Содержит основной класс `App`, который управляет:

- **Окном приложения** - создание и настройка главного окна
- **UI компонентами** - поля ввода, кнопки, метки, прогресс-бар
- **Обработкой событий** - реакции на действия пользователя
- **Процессом сканирования** - запуск и управление сканированием
- **Сохранением результатов** - экспорт результатов в файл

#### Основные методы:

- `NewApp()` - создание нового экземпляра GUI приложения
- `initUI()` - инициализация всех элементов интерфейса
- `setupEventHandlers()` - настройка обработчиков событий
- `startScan()` - запуск процесса сканирования сети
- `saveResults()` - сохранение результатов в файл
- `Run()` - запуск GUI приложения

### cmd/gui/main.go

Точка входа GUI приложения. Содержит минимальный код для запуска:

```go
func main() {
    app := gui.NewApp()
    app.Run()
}
```

## Использование

### Сборка

```bash
go build -o network-scanner-gui ./cmd/gui
```

### Запуск

```bash
./network-scanner-gui
```

### Возможности

- **Автоматическое определение сети** - если поле сети оставлено пустым, приложение автоматически определяет локальную сеть
- **Ручной ввод сети** - можно указать сеть вручную в формате CIDR (например, 192.168.1.0/24)
- **Индикация прогресса** - прогресс-бар показывает ход сканирования
- **Отображение результатов** - режимы `Таблица`/`Карточки`, сортировка и фильтры
- **Подрежимы результатов** - `Devices` / `Security` на вкладке `Сканирование`
- **Сохранение результатов** - возможность сохранить результаты сканирования в текстовый файл
- **Инструменты** - `Ping`, `Traceroute`, `DNS`, `Whois`, `Wi-Fi`, `Wake-on-LAN`, `Аудит портов`, `Risk Signatures`, `Device Control`
- **Operations Center** - история операций инструментов с действиями `Retry`/`Cancel`
- **Host Details Drawer** - панель деталей выбранного хоста + быстрые действия (`Ping`/`Traceroute`/`DNS`/`Whois`/`WOL`)
- **Диагностика сканирования** - копирование/сохранение diagnostics summary последнего запуска
- **Автопрофиль сканирования** - мягкая авто-коррекция heavy-настроек на больших подсетях (`ports/threads`)
- **Рекомендуемые настройки** - отдельная кнопка с безопасным one-click профилем и подсказкой `Почему?` по логике адаптации
- **Бейдж профиля** - строка `Профиль: ...` показывает текущий класс (`small/medium/large/very-large`), класс хранится отдельно в `Preferences` и восстанавливается после перезапуска (с fallback на legacy текст)

## Зависимости

GUI версия использует:
- `fyne.io/fyne/v2` - кроссплатформенный GUI фреймворк
- Внутренние пакеты проекта:
  - `internal/scanner` - логика сканирования
  - `internal/network` - работа с сетью
  - `internal/display` - форматирование результатов

## Технические детали

### Поток выполнения

1. Пользователь запускает приложение
2. Инициализируется GUI интерфейс
3. Пользователь может указать сеть или оставить поле пустым
4. При нажатии "Запустить сканирование":
   - Определяется сеть (автоматически или из поля ввода)
   - Создается экземпляр NetworkScanner
   - Запускается процесс сканирования в отдельной горутине
   - Обновляется прогресс-бар
   - По завершении результаты форматируются и отображаются
5. Пользователь может сохранить результаты в файл

### Обновление UI

Обновления UI выполняются в главном потоке Go для безопасности. Длительные операции (сканирование) выполняются в отдельных горутинах.

### Форматирование результатов

Для вкладки результатов используется комбинированный UI:
- подрежим `Devices` — таблица/карточки + Host Details Drawer + встроенная аналитика;
- подрежим `Security` — сводка findings (`audit` + `risk signatures`) и экспорт HTML отчета.

Для вкладки инструментов используется `RichText` с markdown-совместимым выводом и отдельный `Operations Center` для отслеживания выполнения и повторного запуска операций.

### Адаптивность интерфейса (окно/полный экран)

GUI использует адаптивные профили ширины окна:
- `compact` (<= 1366 px): уплотненная компоновка, `Host Details` переходит в вертикальный split;
- `normal` (1367-2199 px): стандартная компоновка;
- `wide` (>= 2200 px): расширенные ширины таблиц и области результатов.

Поддерживаемая матрица проверок:
- Разрешения: `1366x768`, `1600x900`, `1920x1080`, `1920x1200`, `2560x1440`, `3440x1440`, `3840x2160`.
- Масштаб Windows (DPI): `100%`, `125%`, `150%`, `200%`.
- Режимы: оконный (минимальный/средний) и полноэкранный.

Критерии приемки UI:
- одинаковая доступность функций в оконном и полноэкранном режимах;
- отсутствие обрезки критичных контролов;
- корректное отображение вкладок `Сканирование`, `Топология`, `Инструменты` на `1366x768`.

Быстрая проверка матрицы:
- Windows: `scripts/smoke-gui-resolution.ps1`
- Linux/macOS: `scripts/smoke-gui-resolution.sh`

## Платформы

GUI версия работает на всех платформах, поддерживаемых Fyne:
- Windows
- macOS
- Linux

## Расширение

Для добавления новых функций в GUI:

1. Добавьте новые UI компоненты в метод `initUI()`
2. Добавьте обработчики событий в `setupEventHandlers()`
3. Реализуйте логику в соответствующих методах класса `App`

При необходимости можно создать дополнительные файлы в `internal/gui/` для разделения функциональности (например, отдельный файл для виджетов или для работы с настройками).
