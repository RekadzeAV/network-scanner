## Введение

**Network Scanner** — это кроссплатформенная утилита для сканирования локальных сетей (CLI и GUI). Она помогает обнаруживать устройства в сети, определять открытые порты, протоколы и типы устройств.

### Основные возможности

- 🔍 **Автоматическое определение сети** — утилита сама находит вашу локальную сеть
- 📡 **Обнаружение устройств** — находит все активные устройства в сети
- 🔌 **Сканирование портов** — проверяет открытые TCP порты на устройствах
- 🖥️ **Определение типов устройств** — автоматически определяет тип устройства
- 📊 **Детальная аналитика** — статистика по протоколам, портам и устройствам
- 🏷️ **Определение производителя** — показывает производителя по MAC адресу
- 🖼️ **GUI-режимы результатов** — `Таблица` / `Карточки` с сохранением выбранного режима

## Установка

### Требования

- **Операционная система:** Windows, macOS, Linux (любая архитектура)
- **Go:** версия 1.24 или выше (только для сборки из исходников)
- **Права:** для получения MAC адресов может потребоваться запуск с правами администратора

### Варианты установки

#### Вариант 1: Сборка из исходников (рекомендуется)

1. **Установите Go:**
   ```bash
   # macOS через Homebrew
   brew install go
   
   # Или скачайте с https://go.dev/dl/
   ```

2. **Клонируйте или скачайте проект**

3. **Соберите приложение:**
   ```bash
   cd "Сканер локальной сети"
   go mod download
   go build -o network-scanner ./cmd/network-scanner
   ```

#### Вариант 2: Использование готовых бинарников

Если доступны готовые бинарники для вашей платформы, просто скачайте и запустите.

## Быстрый старт

### Самый простой способ

```bash
# Запустите утилиту без параметров

```

Утилита автоматически:
1. Определит вашу локальную сеть

## Основные функции

### 1. Автоматическое определение сети

Утилита автоматически находит вашу локальную сеть, анализируя активные сетевые интерфейсы.

**Когда использовать вручную:**
- Если у вас несколько сетевых интерфейсов
- Если нужно просканировать другую сеть
- Если автоматическое определение не работает

### 2. Обнаружение активных хостов

Утилита проверяет доступность хостов, пытаясь подключиться к популярным портам:
- 80 (HTTP)
- 443 (HTTPS)
- 22 (SSH)
- 135, 139, 445 (Windows)

### 3. Сканирование портов

Проверяет открытые TCP порты на обнаруженных устройствах. По умолчанию сканирует порты 1-1000.

### 4. Определение типов устройств

Автоматически определяет тип устройства на основе:
- Открытых портов
- Используемых протоколов
- MAC адреса (OUI)

**Поддерживаемые типы:**
- Router/Network Device
- Web Server
- Database Server
- Windows Computer
- Linux/Unix Server
- Printer
- IoT Device

### 5. Аналитика сети

После сканирования выводится детальная статистика:
- Протоколы в сети и их описание
- Используемые порты и их назначение
- Типы устройств
- Общая статистика

## Параметры командной строки

### Основные параметры

| Параметр | Описание | По умолчанию | Пример |
|----------|----------|-------------|--------|
| `--network` | Диапазон сети для сканирования (CIDR) | Автоопределение | `--network 192.168.1.0/24` |
| `--ports` | Порты для сканирования | `1-1000` | `--ports 80,443,8080` |
| `--timeout` | Таймаут TCP/UDP в секундах | `2` | `--timeout 5` |
| `--threads` | Количество потоков | `50` | `--threads 200` |
| `--show-closed` | Показывать закрытые порты | `false` | `--show-closed` |
| `--udp` | Включить UDP-сканирование | `false` | `--udp` |
| `--topology` | Построить топологию после сканирования | `false` | `--topology` |
| `--output-format` | Формат экспорта топологии | пусто | `--output-format json` |
| `--output-file` | Файл для экспорта топологии | авто по формату | `--output-file topology.json` |
| `--snmp-community` | SNMP community (через запятую) | `public` | `--snmp-community public,private` |
| `--snmp-timeout` | Таймаут SNMP в секундах | `2` | `--snmp-timeout 4` |

### Подробное описание

#### `--network`

Указывает сеть для сканирования в формате CIDR.

**Форматы:**
- `192.168.1.0/24` — подсеть с 254 хостами
- `10.0.0.0/16` — большая подсеть
- `172.16.0.0/24` — другая подсеть

**Примеры:**
```bash
./network-scanner --network 192.168.1.0/24
./network-scanner --network 10.0.0.0/16
```

#### `--ports`

Указывает порты для сканирования. Поддерживает несколько форматов.

**Форматы:**
- Диапазон: `1-1000`
- Список: `80,443,8080`
- Комбинация: `80,443,8080-8090,9000`

**Примеры:**
```bash
# Только популярные порты
./network-scanner --ports 80,443,22,3389

# Диапазон портов
./network-scanner --ports 1-1000

# Комбинация
./network-scanner --ports 80,443,8080-8090
```

#### `--timeout`

Таймаут для попытки подключения к порту.

**Форматы:**
- `1` — 1 секунда
- `2` — 2 секунды
- `5` — 5 секунд

**Примеры:**
```bash
# Быстрое сканирование (может пропустить медленные устройства)
./network-scanner --timeout 1

# Медленное, но более точное сканирование
./network-scanner --timeout 10
```

#### `--threads`

Количество одновременных подключений.

**Рекомендации:**
- Быстрая сеть: 200-500
- Обычная сеть: 50 (по умолчанию)
- Медленная сеть: 20-50

**Примеры:**
```bash
# Быстрое сканирование (больше нагрузка на сеть)
./network-scanner --threads 500

# Медленное сканирование (меньше нагрузка)
./network-scanner --threads 50
```

## GUI: работа с результатами

GUI-версия запускается из `cmd/gui` и содержит две ключевые вкладки:

- `Сканирование` — запуск сканирования и просмотр результатов.
- `Топология` — построение и просмотр топологии сети.

### Режимы отображения результатов (вкладка `Сканирование`)

Доступны два режима:

- `Таблица`:
  - колонки `HostName`, `IP`, `MAC`, `Порты`;
  - порты отображаются в виде чипов с переносом строк;
  - при узком окне таблица остается доступной через горизонтальную прокрутку;
  - под таблицей отображаются 2 аналитических блока: `Протоколы` и `Типы устройств`.
- `Карточки`:
  - устройства отображаются в виде карточек (`HostName`, `IP`, `MAC`, порты-чипы);
  - на широком экране показывается сетка из нескольких колонок;
  - на узком экране карточки автоматически перестраиваются в одну колонку;
  - аналитика показывается двумя круговыми диаграммами (протоколы и типы устройств).

### Сохранение режима

Выбранный режим (`Таблица` или `Карточки`) сохраняется в настройках GUI и автоматически восстанавливается после перезапуска приложения.

Также сохраняются параметры отображения результатов:
- сортировка (`IP`/`HostName`);
- лимит отображаемых чипов портов;
- строка фильтра;
- быстрые фильтры по типам устройств;
- флаг `Только с открытыми портами`.

### Практические ограничения

- Для устройств с очень большим числом открытых портов список чипов сокращается, остаток показывается как `+N`.
- Графические диаграммы в карточном режиме кэшируются для ускорения повторной отрисовки.

### Сохранение результатов из GUI

Кнопка `Сохранить результаты` сохраняет именно текущее представление:
- с учетом активных фильтров;
- с учетом выбранной сортировки.

Если после фильтрации список пустой, сохранение не выполняется и показывается информационное сообщение.

## Примеры использования

### Пример 1: Базовое сканирование

```bash
./network-scanner
```

Сканирует автоматически определенную сеть с настройками по умолчанию.

### Пример 2: Сканирование конкретной сети

```bash
./network-scanner --network 192.168.0.0/24
```

Сканирует указанную сеть.

### Пример 3: Сканирование только веб-портов

```bash
./network-scanner --ports 80,443,8080,8443
```

Быстрое сканирование только веб-серверов.

### Пример 4: Быстрое сканирование популярных портов

```bash
./network-scanner --ports 22,80,443,3389,3306 --timeout 1 --threads 200
```

Оптимизировано для быстрого обнаружения основных сервисов.

### Пример 5: Детальное сканирование

```bash
./network-scanner --ports 1-65535 --timeout 5 --threads 50
```

Полное сканирование всех портов (может занять много времени).

### Пример 6: Сканирование с выводом закрытых портов

```bash
./network-scanner --show-closed
```

Показывает также закрытые порты (полезно для отладки).

### Пример 7: Построение топологии сети

```bash
./network-scanner --topology
```

Выполняет сканирование и строит топологию, выводя найденные связи в консоль.

### Пример 8: Экспорт топологии в JSON

```bash
./network-scanner --topology --output-format json --output-file topology.json
```

### Пример 9: Экспорт топологии в GraphML

```bash
./network-scanner --topology --output-format graphml --output-file topology.graphml
```

### Пример 10: Экспорт топологии в PNG/SVG

```bash
./network-scanner --topology --output-format png --output-file topology.png
./network-scanner --topology --output-format svg --output-file topology.svg
```

Для PNG/SVG требуется установленный Graphviz (`dot` в `PATH`).

### Пример 11: Несколько SNMP community и таймаут

```bash
./network-scanner --topology --snmp-community public,private,monitor --snmp-timeout 4
```

### Пример 12: UDP-сканирование вместе с TCP

```bash
./network-scanner --udp --ports 1-256 --timeout 2
```

## Понимание результатов

Ниже описан формат CLI-вывода. В GUI аналогичные данные доступны в двух режимах (`Таблица`/`Карточки`) на вкладке `Сканирование`.

### Таблица результатов

Каждая строка содержит информацию об одном устройстве:

| Колонка | Описание |
|---------|----------|
| **IP** | IP адрес устройства |
| **MAC** | MAC адрес (может быть пустым) |
| **Hostname** | Имя хоста (если доступно) |
| **Порты** | Список открытых портов с протоколом и сервисом |
| **Протоколы** | Используемые протоколы |
| **Тип устройства** | Определенный тип устройства |
| **Производитель** | Производитель по MAC адресу |

### Аналитика

После таблицы выводится аналитика:

#### Протоколы в сети

Показывает какие протоколы используются и на скольких устройствах.

#### Используемые порты

Статистика по портам:
- Сколько устройств используют каждый порт
- Назначение порта
- Название сервиса

#### Типы устройств

Распределение устройств по типам.

#### Общая статистика

- Всего устройств
- Устройств с открытыми портами
- Всего открытых портов
- Уникальных протоколов и портов

## Устранение неполадок

### Проблема: "Не удалось определить локальную сеть"

**Решение:**
```bash
# Укажите сеть вручную
./network-scanner --network 192.168.1.0/24
```

### Проблема: MAC адреса не определяются

**Причины:**
- Нет прав администратора
- Устройства не отвечают на ARP запросы
- Сетевая конфигурация блокирует ARP

**Решение:**
- Запустите с правами администратора:
  ```bash
  sudo ./network-scanner  # Linux/macOS
  ```
- Это нормально, остальная функциональность работает

### Проблема: Устройства не обнаруживаются

**Возможные причины:**
- Устройства не отвечают на проверяемые порты
- Firewall блокирует подключения
- Устройства в другой подсети

**Решение:**
- Увеличьте таймаут: `--timeout 10`
- Проверьте firewall настройки
- Убедитесь, что устройства в той же сети

### Проблема: Сканирование слишком медленное

**Решение:**
```bash
# Уменьшите диапазон портов
./network-scanner --ports 80,443,22

# Увеличьте количество потоков
./network-scanner --threads 200

# Уменьшите таймаут (осторожно, может пропустить устройства)
./network-scanner --timeout 1
```

### Проблема: Слишком много ложных срабатываний

**Решение:**
```bash
# Увеличьте таймаут для более точного определения
./network-scanner --timeout 5
```

## Часто задаваемые вопросы

### Q: Можно ли сканировать UDP порты?

**A:** Да. Используйте флаг `--udp`. По умолчанию приложение сканирует TCP; с `--udp` добавляется проверка популярных UDP-портов.

### Q: Нужны ли права администратора?

**A:** Для получения MAC адресов — да, для остального функционала — нет.

### Q: Можно ли сохранить результаты в файл?

**A:** Используйте перенаправление вывода:
```bash
./network-scanner > results.txt
```

### Q: Как сканировать только определенные устройства?

**A:** Укажите конкретную подсеть или используйте фильтрацию результатов.

### Q: Влияет ли сканирование на производительность сети?

**A:** Да, но обычно минимально. Уменьшите `--threads`, если сеть перегружена.

### Q: Можно ли использовать в корпоративной сети?

**A:** Убедитесь, что у вас есть разрешение на сканирование сети. Некоторые корпоративные политики запрещают сетевые сканеры.

### Q: Как интерпретировать результаты?

**A:** 
- Открытые порты показывают доступные сервисы
- Тип устройства помогает понять назначение устройства
- Протоколы показывают используемые технологии
- Аналитика помогает понять общую картину сети
