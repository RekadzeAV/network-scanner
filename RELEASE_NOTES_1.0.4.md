# Release Notes v1.0.4

**Дата релиза:** 2026-03-27  
**Версия:** 1.0.4

## Основные изменения

### 1) Повышение точности топологии

- Добавлена модель источника и достоверности связи:
  - `SourceType`: `lldp`, `fdb`, `inferred`
  - `Confidence`: `high`, `medium`, `low`
  - `Evidence`: краткая причина формирования связи
- Улучшена дедупликация связей:
  - приоритет LLDP над FDB для одного endpoint набора
  - выбор наиболее достоверной связи при конфликте

### 2) Исправление фильтрации MAC в FDB

- Переписана логика определения multicast/broadcast MAC:
  - multicast определяется по I/G биту первого байта
  - broadcast `ff:ff:ff:ff:ff:ff` игнорируется
  - `00:00:00:00:00:00` игнорируется
  - self-MAC коммутатора исключается

Результат: меньше ложных связей в топологии.

### 3) Формализованный SNMP отчет

- Добавлен расширенный API `CollectWithReport(...)`:
  - `TotalSNMPTargets`
  - `Connected`
  - `Partial`
  - `Failed`
  - список `Failures` с типами ошибок
- В CLI и GUI добавлен вывод SNMP summary:
  - видно, сколько устройств опросилось полностью,
  - сколько частично,
  - сколько не подключилось.

### 4) Повышение устойчивости GUI

- Во время длительного SNMP+Build этапа временно блокируются:
  - построение топологии,
  - сохранение,
  - refresh/open preview.
- После завершения или ошибки состояние кнопок корректно восстанавливается.

### 5) Regression smoke-проверки

Добавлены скрипты для быстрых проверок CLI-сценариев:

- Без топологии:
  - `scripts/smoke-cli-no-topology.sh`
  - `scripts/smoke-cli-no-topology.ps1`
- С топологией:
  - `scripts/smoke-cli-topology.sh`
  - `scripts/smoke-cli-topology.ps1`

Smoke-сценарии используют `127.0.0.1/32` и короткий диапазон портов для быстрого прогона.

### 6) Синхронизация документации

Обновлены документы:

- `README.md`
- `docs/USER_GUIDE.md`
- `docs/TECHNICAL.md`
- `docs/ARCHITECTURE.md`
- `CHANGELOG.md`

Документация теперь отражает актуальные флаги CLI, topology flow и SNMP reporting.

## Технические заметки

- Обратная совместимость базового пути (без `--topology`) сохранена.
- Ключевые тесты модулей топологии и SNMP проходят.
- Сборка CLI/GUI проходит успешно.

## Проверки перед публикацией

Рекомендуемый минимальный чек-лист:

1. `go test ./...`
2. `go build ./cmd/network-scanner`
3. `go build ./cmd/gui`
4. smoke-прогоны:
   - `smoke-cli-no-topology`
   - `smoke-cli-topology`

## Известные ограничения

- Точность топологии зависит от доступности SNMP и качества LLDP/FDB данных устройства.
- Для экспорта PNG/SVG требуется установленный Graphviz (`dot` в `PATH`).
