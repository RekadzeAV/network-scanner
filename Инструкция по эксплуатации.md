## Общие сведения

**Network Scanner** — это кроссплатформенная утилита для сканирования локальных сетей, которая позволяет обнаруживать активные устройства, сканировать открытые порты, определять типы устройств и получать детальную аналитику о состоянии сети.

Программа доступна в двух версиях:
- **CLI версия** — командная строка для опытных пользователей
- **GUI версия** — графический интерфейс для удобного использования

---

## Назначение программы

Network Scanner предназначен для:

- 🔍 **Обнаружения устройств** в локальной сети
- 📡 **Сканирования портов** на обнаруженных устройствах
- 🖥️ **Определения типов устройств** (роутеры, серверы, принтеры и т.д.)
- 📊 **Получения аналитики** о протоколах и сервисах в сети
- 🏷️ **Определения производителей** устройств по MAC адресам
- 📋 **Формирования отчетов** о состоянии сети

**Важно:** Программа предназначена для использования только в ваших собственных сетях или сетях, где у вас есть явное разрешение на сканирование.

- **Память:** 50 МБ свободной оперативной памяти

## Установка программы

### Вариант 1: Использование готового исполняемого файла

1. Скачайте исполняемый файл для вашей платформы
2. Для Windows: запустите `network-scanner.exe` (CLI) или `network-scanner-gui.exe` (GUI)
3. Для Linux/macOS: сделайте файл исполняемым и запустите:
   ```bash
   chmod +x network-scanner
   ./network-scanner
   ```

### Вариант 2: Сборка из исходников

#### Требования для сборки

- Установленный Go версии 1.21 или выше
- Git (для клонирования репозитория)

#### Шаги установки

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/RekadzeAV/network-scanner.git
   cd network-scanner
   ```

2. **Установите зависимости:**
   ```bash
   go mod download
   ```

3. **Соберите CLI версию:**
   ```bash
   go build -o network-scanner ./cmd/network-scanner
   ```

4. **Соберите GUI версию:**
   ```bash
   go build -o network-scanner-gui ./cmd/gui
   ```

#### Использование скриптов сборки

**Windows:**
```cmd
scripts\build.bat
```

**Linux/Unix:**
```bash
./scripts/build.sh
```

**macOS:**
```bash
./scripts/build-macos.sh
```

## Использование CLI версии

### Базовое использование

Самый простой способ запуска — без параметров:

```bash
./network-scanner
```

Программа автоматически:
1. Определит вашу локальную сеть
2. Найдет все активные устройства
3. Просканирует порты 1-1000 на каждом устройстве
4. Выведет результаты в виде таблицы

### Справка по использованию

Для просмотра справки используйте:

```bash
./network-scanner --help
```

Для просмотра версии:

```bash
./network-scanner --version
```

### Прерывание сканирования

Для остановки сканирования нажмите `Ctrl+C`. Программа корректно завершит работу и выведет уже полученные результаты.

## Использование GUI версии

### Интерфейс программы

GUI версия предоставляет удобный графический интерфейс с следующими элементами:

1. **Поле ввода сети** — для указания сети в формате CIDR (например, `192.168.1.0/24`)
   - Если оставить пустым, сеть определится автоматически
2. **Кнопка "Запустить сканирование"** — запускает процесс сканирования
3. **Кнопка "Сохранить результаты"** — сохраняет результаты в файл (активна после завершения сканирования)
4. **Индикатор статуса** — показывает текущее состояние программы
5. **Прогресс-бар** — отображает ход выполнения сканирования
6. **Область результатов** — прокручиваемая область с результатами сканирования

### Работа с программой

1. **Запустите программу:**
   ```bash
   ./network-scanner-gui
   ```

2. **Укажите сеть (опционально):**
   - Оставьте поле пустым для автоматического определения
   - Или введите сеть в формате CIDR: `192.168.1.0/24`

3. **Запустите сканирование:**
   - Нажмите кнопку "Запустить сканирование"
   - Дождитесь завершения (прогресс отображается в прогресс-баре)

4. **Просмотрите результаты:**
   - Результаты отображаются в области результатов
   - Используйте прокрутку для просмотра всех результатов

5. **Сохраните результаты (опционально):**
   - Нажмите кнопку "Сохранить результаты"
   - Выберите место сохранения файла
   - Результаты будут сохранены в текстовом формате

## Параметры командной строки

### Основные параметры

| Параметр | Описание | По умолчанию | Пример |
|----------|----------|--------------|--------|
| `--network` | Диапазон сети для сканирования (CIDR) | Автоопределение | `--network 192.168.1.0/24` |
| `--ports` | Порты для сканирования | `1-1000` | `--ports 80,443,8080` |
| `--timeout` | Таймаут для подключения (в секундах) | `3` | `--timeout 5` |
| `--threads` | Количество потоков | `100` | `--threads 200` |
| `--show-closed` | Показывать закрытые порты | `false` | `--show-closed` |
| `--help` | Показать справку | - | `--help` |
| `--version` | Показать версию | - | `--version` |

### Подробное описание параметров

#### `--network` (Диапазон сети)

Указывает сеть для сканирования в формате CIDR (Classless Inter-Domain Routing).

**Форматы:**
- `192.168.1.0/24` — подсеть с 254 хостами (192.168.1.1 - 192.168.1.254)
- `10.0.0.0/16` — большая подсеть с 65534 хостами
- `172.16.0.0/24` — другая подсеть

**Примеры:**
```bash
./network-scanner --network 192.168.1.0/24
./network-scanner --network 10.0.0.0/16
```

**Примечание:** Если параметр не указан, программа автоматически определит локальную сеть.

#### `--ports` (Диапазон портов)

Указывает порты для сканирования. Поддерживает несколько форматов.

**Форматы:**
- Диапазон: `1-1000` — все порты от 1 до 1000
- Список: `80,443,8080` — только указанные порты
- Комбинация: `80,443,8080-8090,9000` — комбинация списка и диапазона

**Примеры:**
```bash
# Только популярные порты
./network-scanner --ports 80,443,22,3389

# Диапазон портов
./network-scanner --ports 1-1000

# Комбинация
./network-scanner --ports 80,443,8080-8090
```

**Рекомендации:**
- Для быстрого сканирования: `80,443,22,3389`
- Для стандартного сканирования: `1-1000` (по умолчанию)
- Для полного сканирования: `1-65535` (может занять много времени)

#### `--timeout` (Таймаут)

Таймаут для попытки подключения к порту в секундах.

**Примеры:**
```bash
# Быстрое сканирование (может пропустить медленные устройства)
./network-scanner --timeout 1

# Медленное, но более точное сканирование
./network-scanner --timeout 10
```

**Рекомендации:**
- Быстрая сеть: `1-3`
- Обычная сеть: `3` (по умолчанию)
- Медленная сеть или удаленные устройства: `5-10`

#### `--threads` (Количество потоков)

Количество одновременных подключений при сканировании.

**Рекомендации:**
- Быстрая сеть: 200-500 потоков
- Обычная сеть: 100 потоков (по умолчанию)
- Медленная сеть: 50 потоков
- Максимальное значение: 1000 (не рекомендуется превышать)

**Примеры:**
```bash
# Быстрое сканирование (больше нагрузка на сеть)
./network-scanner --threads 500

# Медленное сканирование (меньше нагрузка)
./network-scanner --threads 50
```

**Предупреждение:** Большое количество потоков может привести к перегрузке сети или блокировке со стороны firewall.

#### `--show-closed` (Показывать закрытые порты)

По умолчанию программа показывает только открытые порты. Этот флаг включает отображение закрытых портов.

**Использование:**
```bash
./network-scanner --show-closed
```

**Применение:** Полезно для отладки и детального анализа состояния портов.

- SSH (22)

```bash
./network-scanner --ports 1-65535 --timeout 5 --threads 50
```

Сохраняет результаты сканирования в файл `scan_results.txt`.

## Интерпретация результатов

### Структура вывода

Программа выводит результаты в два этапа:

1. **Процесс сканирования** — информация о ходе выполнения
2. **Результаты сканирования** — таблица с найденными устройствами
3. **Аналитика** — статистика по сети

### Таблица результатов

Каждая строка таблицы содержит информацию об одном устройстве:

| Колонка | Описание | Пример |
|---------|----------|--------|
| **IP** | IP адрес устройства | `192.168.1.1` |
| **MAC** | MAC адрес устройства | `aa:bb:cc:dd:ee:ff` |
| **Hostname** | Имя хоста (если доступно) | `router.local` |
| **Порты** | Список открытых портов с протоколом и сервисом | `80/tcp (HTTP)`, `443/tcp (HTTPS)` |
| **Протоколы** | Используемые протоколы | `HTTP`, `HTTPS` |
| **Тип устройства** | Определенный тип устройства | `Router/Network Device` |
| **Производитель** | Производитель по MAC адресу | `VMware`, `Apple` |

### Типы устройств

Программа автоматически определяет тип устройства на основе открытых портов:

- **Router/Network Device** — роутеры и сетевое оборудование (порты 80, 443, 22)
- **Web Server** — веб-серверы (порты 80, 443, 8080, 8443)
- **Database Server** — серверы баз данных (порты 3306, 5432, 1433)
- **Windows Computer** — компьютеры Windows (порты 3389, 445)
- **Linux/Unix Server** — серверы Linux/Unix (порт 22)
- **Printer** — принтеры (порты 9100, 515)
- **IoT Device** — устройства Интернета вещей (мало портов)
- **Unknown Device** — неизвестный тип устройства

### Аналитика

После таблицы выводится детальная аналитика:

#### Протоколы в сети

Показывает какие протоколы используются и на скольких устройствах:
- HTTP — на 5 устройствах
- HTTPS — на 3 устройствах
- SSH — на 2 устройствах

#### Используемые порты

Статистика по портам:
- Сколько устройств используют каждый порт
- Назначение порта
- Название сервиса

Пример:
- Порт 80 (HTTP) — используется на 5 устройствах
- Порт 443 (HTTPS) — используется на 3 устройствах
- Порт 22 (SSH) — используется на 2 устройствах

#### Типы устройств

Распределение устройств по типам:
- Router/Network Device — 2 устройства
- Web Server — 3 устройства
- Windows Computer — 5 устройств

#### Общая статистика

- Всего устройств: 12
- Устройств с открытыми портами: 10
- Всего открытых портов: 45
- Уникальных протоколов: 5
- Уникальных портов: 15

### Понимание состояний портов

- **open** — порт открыт и доступен
- **closed** — порт закрыт (показывается только с флагом `--show-closed`)
- **filtered** — порт фильтруется firewall (в текущей версии не определяется)

## Устранение неполадок

### Проблема: "Не удалось определить локальную сеть"

**Симптомы:**
```
Ошибка: не удалось определить локальную сеть: ...
Укажите сеть вручную с помощью параметра --network
```

**Причины:**
- Нет активных сетевых интерфейсов
- Несколько сетевых интерфейсов
- Нестандартная сетевая конфигурация

**Решение:**
```bash
# Укажите сеть вручную
./network-scanner --network 192.168.1.0/24
```

### Проблема: MAC адреса не определяются

**Симптомы:**
- В таблице результатов колонка MAC пустая
- Сообщения об ошибках при получении MAC адресов

**Причины:**
- Нет прав администратора/root
- Устройства не отвечают на ARP запросы
- Сетевая конфигурация блокирует ARP
- Устройства находятся в другой подсети

**Решение:**
- **Linux/macOS:** Запустите с правами root:
  ```bash
  sudo ./network-scanner
  ```
- **Windows:** Запустите от имени администратора
- Это нормально, остальная функциональность работает без MAC адресов

### Проблема: Устройства не обнаруживаются

**Симптомы:**
- Результаты сканирования не найдены
- Найдено 0 активных хостов

**Возможные причины:**
- Устройства не отвечают на проверяемые порты (80, 443, 22, 135, 139, 445)
- Firewall блокирует подключения
- Устройства в другой подсети
- Неверно указан диапазон сети

**Решение:**
1. Увеличьте таймаут:
   ```bash
   ./network-scanner --timeout 10
   ```
2. Проверьте firewall настройки на устройствах
3. Убедитесь, что устройства в той же сети
4. Проверьте правильность указания сети:
   ```bash
   ./network-scanner --network 192.168.1.0/24
   ```

### Проблема: Сканирование слишком медленное

**Симптомы:**
- Сканирование занимает очень много времени
- Низкая скорость обработки

**Решение:**
```bash
# Уменьшите диапазон портов
./network-scanner --ports 80,443,22

# Увеличьте количество потоков
./network-scanner --threads 200

# Уменьшите таймаут (осторожно, может пропустить устройства)
./network-scanner --timeout 1
```

### Проблема: Слишком много ложных срабатываний

**Симптомы:**
- Обнаруживаются устройства, которых нет
- Неправильное определение типов устройств

**Решение:**
```bash
# Увеличьте таймаут для более точного определения
./network-scanner --timeout 5
```

### Проблема: GUI версия не запускается

**Симптомы:**
- Ошибка при запуске GUI версии
- Окно не открывается

**Причины:**
- Отсутствует графический интерфейс
- Проблемы с зависимостями Fyne
- Несовместимость с системой

**Решение:**
1. Убедитесь, что система имеет графический интерфейс
2. Пересоберите GUI версию:
   ```bash
   go build -o network-scanner-gui ./cmd/gui
   ```
3. Используйте CLI версию как альтернативу

### Проблема: Ошибки при сборке

**Симптомы:**
- Ошибки компиляции
- Отсутствующие зависимости

**Решение:**
1. Обновите зависимости:
   ```bash
   go mod download
   go mod tidy
   ```
2. Убедитесь, что используется Go версии 1.24 или выше:
   ```bash
   go version
   ```
3. Проверьте наличие всех зависимостей в `go.mod`

### Проблема: Программа завершается с ошибкой

**Симптомы:**
- Программа аварийно завершается
- Сообщения об ошибках

**Решение:**
1. Проверьте параметры командной строки
2. Убедитесь, что указана правильная сеть
3. Проверьте права доступа к сети
4. Запустите с правами администратора (для получения MAC адресов)

## Ограничения и предупреждения

### Технические ограничения

1. **Только TCP порты:** Текущая версия поддерживает сканирование только TCP портов. UDP сканирование не поддерживается.

2. **MAC адреса:** Получение MAC адресов может требовать прав администратора и работает не на всех системах одинаково.

3. **ARP запросы:** ARP запросы могут не работать без прав администратора или в некоторых сетевых конфигурациях.

4. **Определение устройств:** Определение типа устройства основано на эвристике (открытые порты) и может быть неточным.

5. **Производители:** База производителей по MAC адресам (OUI) ограничена. Многие производители могут определяться как "Unknown".

### Безопасность и правовые аспекты

⚠️ **ВАЖНО:** 

1. **Используйте только в своих сетях:** Программа предназначена для использования только в ваших собственных сетях или сетях, где у вас есть явное разрешение на сканирование.

2. **Не используйте для несанкционированного сканирования:** Сканирование сетей без разрешения может быть незаконным и нарушать политики безопасности.

3. **Корпоративные сети:** Убедитесь, что у вас есть разрешение на сканирование корпоративной сети. Некоторые корпоративные политики запрещают использование сетевых сканеров.

4. **Влияние на сеть:** Сканирование создает нагрузку на сеть. Используйте разумное количество потоков, чтобы не перегружать сеть.

5. **Firewall и IDS:** Сканирование может быть обнаружено системами обнаружения вторжений (IDS) и заблокировано firewall.

### Рекомендации по использованию

1. **Начните с малого:** Используйте небольшой диапазон портов для первого сканирования
2. **Используйте разумные таймауты:** Не устанавливайте слишком маленькие таймауты
3. **Контролируйте нагрузку:** Не используйте слишком много потоков одновременно
4. **Сохраняйте результаты:** Сохраняйте результаты для последующего анализа
5. **Регулярное сканирование:** Регулярное сканирование помогает отслеживать изменения в сети
